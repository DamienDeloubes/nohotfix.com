# Implementation Plan: Create Spec

**Branch**: `011-create-spec` | **Date**: 2026-03-09 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/011-create-spec/spec.md`

## Summary

Implement the first Authoring-domain feature: Admin users can create new specs in the Spec Library via a form (POST endpoint + React form). The spec_library table already exists (001 migration). Work involves implementing the `createLibrarySpec` use-case, the Kysely repository adapter, the POST API endpoint, changelog integration via the Audit domain, and the frontend form + minimal detail page. The "system under test" combobox requires a GET endpoint for distinct values. No schema migration is needed.

## Technical Context

**Language/Version**: TypeScript 5.7, Node.js 20
**Primary Dependencies**: Fastify 5, Kysely, TanStack Router + Query, React, Zod, TipTap (rich text)
**Storage**: PostgreSQL — existing `spec_library` and `changelog` tables (no migration)
**Testing**: Vitest (unit + integration), Playwright (E2E)
**Target Platform**: Linux server (API), Web browser (SPA)
**Project Type**: Web application (modular monolith)
**Performance Goals**: API write <1000ms p95 (constitution), form submit perceived <2s
**Constraints**: org_id tenancy on all queries, Admin-only access (owner/admin role)
**Scale/Scope**: B2B SaaS, <1000 orgs in v1, specs per org likely <500
