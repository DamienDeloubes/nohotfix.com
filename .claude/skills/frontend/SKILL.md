# Frontend Development — NoHotfix

TRIGGER: When building, modifying, or debugging pages, components, forms, hooks, or routes in `apps/app/`.

## Stack

- React 18 + Vite 6 + TypeScript 5.7
- TanStack Router v1.91 (file-based routing)
- TanStack Query v5 (server state)
- react-hook-form v7 + Zod v3 (`@hookform/resolvers/zod`)
- Tailwind CSS 3
- No component library — raw HTML elements + inline styles currently

## Project Structure

## Routing Conventions

- File-based routing: route files under `apps/app/src/routes/` map to URL paths
- Every route file exports `const Route = createFileRoute('/<path>')({ component: PageName })`
- Layout routes use `_` prefix: `_authenticated.tsx`
- Dynamic params use `$`: `$orgSlug`, `$runId`, `$playbookId`, `$specId`
- Route guards use `beforeLoad` with `throw redirect({ to, params })`
- Named function components (not arrow exports):

  ```tsx
  export const Route = createFileRoute('/_authenticated/$orgSlug/playbooks/')({
    component: PlaybooksPage,
  });

  function PlaybooksPage() {
    return <div>...</div>;
  }
  ```

- Access route params via `Route.useParams()`
- `routeTree.gen.ts` is auto-generated by `TanStackRouterVite()` plugin — never edit manually

## Data Fetching

- **API base URL**: `import.meta.env.VITE_API_URL` (fallback: `https://api.nohotfix.com`)
- **Auth tokens**: `getAccessToken()` from `lib/session.ts` — returns cached token or refreshes
- **Cookie-based endpoints** (`/auth/refresh`, `/auth/logout`): use `credentials: 'include'`
- **Authenticated endpoints** (`/api/*`): use `Authorization: Bearer ${token}` header
- **ApiClient** (`@nohotfix/api-client`): typed fetch wrapper with 401 retry, Zod validation, injected via React context

  ```tsx
  import { useApiQuery } from '@nohotfix/api-client';

  // Domain hooks use useApiQuery/useApiMutation — no manual fetch needed
  ```

- **Query key factories** (`api/query-keys.ts`): hierarchical keys scoped by `orgSlug`
  ```tsx
  orgKeys.checkSlug(slug); // ['check-slug', slug]
  settingsKeys.org(orgSlug); // ['settings', orgSlug, 'org']
  runKeys.list(orgSlug); // ['runs', orgSlug, 'list']
  runKeys.detail(orgSlug, id); // ['runs', orgSlug, 'detail', id]
  ```
- **Query key policy (HARD CONSTRAINT)**: ALL `queryKey` values and mutation invalidation
  keys MUST be defined in `apps/app/src/api/query-keys.ts`. Inline string-literal query
  keys are forbidden. Keys are organised by bounded context and aggregate (e.g.,
  `orgKeys`, `settingsKeys`, `runKeys`, `playbookKeys`). Domain UI hooks in
  `packages/domains/<ctx>/src/ui/hooks/` MUST accept `queryKey` (and `invalidateKeys`
  for mutations) as options parameters — they cannot import from `apps/app`. Route files
  pass centralised key factories to hooks at call-site:

  ```tsx
  // In apps/app/src/routes/.../general.tsx
  import { orgKeys, settingsKeys } from '../../../../api/query-keys.js';

  const { data } = useOrganisationDetails({
    orgSlug,
    queryKey: settingsKeys.org(orgSlug),
  });
  const { mutateAsync } = useRenameOrganisation({
    orgSlug,
    invalidateKeys: [settingsKeys.org(orgSlug), orgKeys.userOrgs()],
  });
  ```

- **QueryClient defaults**: `staleTime: 30_000ms`, `retry: 1`
- **Session**: `staleTime: 5min`, `retry: false`
- Invalidate cache on mutation success — always use centralised keys:
  ```tsx
  onSuccess: () => {
    void queryClient.invalidateQueries({ queryKey: orgKeys.userOrgs() });
  };
  ```

## Form Patterns

- Always use Zod schemas from `@nohotfix/shared` — never duplicate schemas
- Standard setup:
  ```tsx
  const { register, handleSubmit, watch, formState: { errors, isSubmitting } } = useForm<T>({
    resolver: zodResolver(SharedSchema),
    defaultValues: { ... },
  });
  ```
- Debounced validation queries: `useState` + `useEffect` with `setTimeout(300ms)` + `useQuery({ enabled })`
- Mutations via domain hooks (which use `useApiMutation` from `@nohotfix/api-client`) with `mutateAsync` in `handleSubmit`
- Inline error display: `{errors.field && <p style={{ color: 'red' }}>{errors.field.message}</p>}`
- Disabled submit when invalid or submitting

## Component Conventions

- Named function components (PascalCase)
- Props interfaces defined directly above the component
- Inline styles currently
- No `import React` needed (`jsx: react-jsx` transform)
- Use `.js` extensions in relative imports (ESM resolution)

## Import Rules

- Types and Zod schemas: `import { Schema, Type } from '@nohotfix/shared'`
- Domain identity package: `import { ... } from '@nohotfix/domain-identity'`
- Path alias: `@/*` maps to `./src/*`
- Never import from `apps/api` or `packages/db` — frontend only consumes `@nohotfix/shared`

## Environment Variables

- `VITE_API_URL` — Backend API (default: `https://api.nohotfix.com`)
- `VITE_WEB_URL` — Marketing/auth site (default: `https://nohotfix.com`)
- Set in `apps/app/.env.local` (gitignored)

## State Management

- **Server state**: TanStack Query (SSOT for all API data)
- **Client state**: minimal — UI state + form state via react-hook-form
- **No global store** (no Redux/Zustand) — TanStack Query cache is the source of truth
- **Auth session**: in-memory access token + httpOnly refresh cookie (managed by `lib/session.ts`, exposed as `tokenManager` to `ApiClient`)
- **Polling intervals**: 5s for active runs, 10s for dashboard, no polling for history (planned)

## Navigation

- Org switching: `navigate({ to: '/$orgSlug', params: { orgSlug } })`
- All authenticated routes are org-scoped under `/$orgSlug/`
- After form success: callback prop triggers navigation
